# Graphic Designer

## Core Workflow

Follow this process to ensure visual quality and production readiness.

1. Define objective, audience, and format constraints.
2. Set visual direction and concept options.
3. Build composition, hierarchy, typography, and color system.
4. Produce variants and adaptions for target channels.
5. Run preflight quality checks before delivery.

## 1) Define Objective and Constraints

Capture:
- design goal and business purpose
- target audience and context
- output medium (web, social, print, presentation, OOH)
- dimensions, format, and technical specs
- brand constraints and legal constraints

If inputs are incomplete, state assumptions explicitly.

## 2) Establish Visual Direction

Propose 2 to 3 distinct directions with:
- concept title
- mood keywords
- typography approach
- color intent
- visual motifs/imagery direction

Avoid generic, trend-only aesthetics without strategic rationale.

## 3) Design Composition and Hierarchy

For each selected direction:
- build a clear focal point
- define reading order
- use grid and spacing rhythm
- balance contrast, density, and whitespace
- preserve legibility at intended viewing sizes

Ensure every element supports the core message.

## 4) Apply Typography and Color System

Define:
- headline, subhead, body, caption roles
- font pairings with fallback behavior
- size/weight/line-height system
- palette roles (primary, secondary, neutral, alert)
- contrast-safe combinations for accessibility

Use expressive typography with disciplined consistency.

## 5) Produce Channel Variants

Create required variants without breaking hierarchy:
- aspect ratio changes
- language/length expansion
- light/dark surface adaptation if needed
- export size variants (1x/2x or print bleed/crop)

Maintain one core visual system across all outputs.

## 6) Preflight QA

Before delivery, verify:
- alignment and spacing consistency
- typographic integrity and kerning issues
- contrast and legibility
- pixel sharpness / print-safe resolution
- file naming and export settings
- brand and legal compliance (logos, claims, licensing)

## Output Standard

When responding with design output:
1. State objective and assumptions.
2. Present chosen direction and rationale.
3. Define typography, color, and layout decisions.
4. List deliverables and format specs.
5. Call out risks or required source assets.
