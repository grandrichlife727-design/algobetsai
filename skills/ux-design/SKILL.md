# Ux Design

## Core Workflow

Follow this sequence to keep UX decisions grounded in user outcomes.

1. Frame the problem and success criteria.
2. Define users, jobs, and constraints.
3. Map journeys and task flows.
4. Design information architecture and interaction model.
5. Produce wireframes and interaction notes.
6. Validate with heuristics and test scenarios.

## 1) Frame Problem and Success

State:
- primary user outcome
- business outcome
- measurable success criteria
- known constraints (time, platform, compliance, technical)

If goals conflict, prioritize user-critical task completion first.

## 2) Define Users and Jobs

Capture lightweight persona inputs:
- who they are
- context of use
- core jobs-to-be-done
- pain points and failure moments

Avoid broad demographic profiles when behavioral context is enough.

## 3) Map Journeys and Task Flows

Map end-to-end flow:
- entry points
- decision branches
- system feedback
- recovery/error paths
- completion confirmation

Include alternate paths for first-time and returning users when behavior differs.

## 4) Design IA and Interaction Model

Define:
- navigation structure
- content grouping and labeling
- page/screen hierarchy
- key interaction patterns (search, filter, forms, confirmations)

Prefer predictable patterns for high-frequency tasks; use novel patterns only with clear payoff.

## 5) Produce Wireframes and Behavior Notes

For each key screen/state, define:
- intent of the screen
- primary action and secondary actions
- input and validation behavior
- empty/loading/error/success states
- accessibility considerations (focus order, labels, keyboard actions)

Keep wireframes structurally clear before visual polish.

## 6) Validate Before Handoff

Run a fast validation pass:
- heuristic review (status visibility, match to real-world language, error prevention, consistency)
- cognitive load review (number of choices, clarity of next step, progressive disclosure)
- scenario walkthroughs for top 3 user tasks
- edge-case walkthroughs for failure and recovery

## Output Standard

When delivering UX output:
1. Provide problem framing and assumptions first.
2. Include user/task flow in numbered steps.
3. Describe IA decisions and why.
4. Document states and edge cases.
5. End with validation findings and open risks.
